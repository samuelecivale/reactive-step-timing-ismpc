
Dear Professor Scianca,

we would like to give you a short update before preparing the final presentation with the professor.

We extended the provided IS-MPC humanoid locomotion workspace with a **reactive step adaptation layer**.
The extension can modify online:

- the timing of the next step
- the position of the next foothold

in response to an external perturbation.

The implementation was integrated into the existing pipeline while keeping the baseline MPC structure as unchanged as possible.

Main additions include:

- a new `StepTimingAdapter` module
- separation between nominal and active footstep plan
- online swing-foot re-anchoring for consistency after plan updates
- CLI-based perturbation scheduling
- reproducible headless testing with JSON summaries
- basic qualitative force visualization in the viewer

We tested two perturbation families:

1. **body push** (`push_target = base`)
2. **stance-foot / slip-like perturbation** (`push_target = stance_foot`)

After several exploratory batteries, we tuned the reactive-layer parameters and focused the final quantitative comparison on the **body-push scenario**, which gave the cleanest and most interpretable results.

The clearest final result is in the body-push case at `push_phase = 0.55`.

With the final tuning:

- the baseline controller fails at lower force levels
- the adapted controller survives at higher force levels before failing

So the adaptive layer shifts the failure frontier forward in the body-push scenario.

We also implemented and ran slip-like stance-foot perturbation tests.
However, in our setup those results were less conclusive than the body-push ones, likely because the stance-foot scenario is much more sensitive to the simulator contact/friction model.

For this reason, we currently treat the slip-like tests as an exploratory extension rather than as the main claim of the project.

At this point we have:

- a final simulation file
- a reproducible headless testing workflow
- final quantitative body-push results
- qualitative viewer runs and diagnostic plots
- an updated README summarizing the implementation and experiments
